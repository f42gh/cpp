# 配列の基本と2次元配列

## 1次元配列とは

メモリ上に値が一列に並んだもの。インデックスで要素を取り出す。

```
A[0]  A[1]  A[2]  A[3]  A[4]
  3     7     1     9     2
```

```cpp
int A[5] = {3, 7, 1, 9, 2};
cout << A[2];  // 1
```

競プロでは固定サイズより `vector<int>` を使うことが多い（→ [vector.md](vector.md)）。

---

## 2次元配列とは

「配列の配列」。表・グリッドのイメージ。`A[行][列]` でアクセスする。

```
      j=0  j=1  j=2
i=0 [  3    7    1  ]
i=1 [  9    2    5  ]
i=2 [  4    8    6  ]
```

```cpp
int A[3][3] = {{3,7,1},{9,2,5},{4,8,6}};
cout << A[1][2];  // 5
```

固定サイズで宣言すると**全行が同じ列数**になる（長方形）。

---

## スタックサイズの限界

ローカル変数（関数の中で宣言した配列）はスタックに乗る。スタックは通常 **8MB** しかない。

| 宣言 | メモリ使用量 | 結果 |
|---|---|---|
| `int A[1000]` | 4KB | 問題なし |
| `int A[200000]` | 800KB | ギリギリOK |
| `int A[10000][10000]` | 約400MB | **セグフォ** |

グローバル変数（`main` の外）に置けばスタックを使わないので大きくできる。

```cpp
int A[200000];  // グローバル → OK

int main() {
    // int A[200000]; ← ローカルだと危険
}
```

## ジャグ配列（行ごとに長さが違う2次元）

今回のような「各行の長さ `L_i` がバラバラ」なケース：

```
行0: [3, 7]          // 長さ2
行1: [9, 2, 5, 1]    // 長さ4
行2: [4]             // 長さ1
```

固定サイズ `int A[N][MAX_L]` だと無駄が多く、制約次第でメモリが死ぬ。
→ `vector<vector<int>>` を使う。

```cpp
int n;
cin >> n;
vector<vector<int>> A(n);   // n行の空のvector

for (int i = 0; i < n; i++) {
    int l;
    cin >> l;
    A[i].resize(l);         // i行目をl個分確保
    for (int j = 0; j < l; j++) cin >> A[i][j];
}

int x, y;
cin >> x >> y;
cout << A[x-1][y-1];        // 問題文は1-indexed なので -1 する
```

**なぜ `int A[N][MAX_L]` が無理か（ABC457B の例）**

- N ≤ 2×10⁵、L の総和 ≤ 2×10⁵
- `int A[200000][200000]` は **約400MB** → アウト
- 実際に必要なのはLの総和 = 2×10⁵ 個だけ
- `vector<vector<int>>` なら必要な分だけ確保できる

## よくある間違い

```cpp
// NG: vectorのサイズをx,yで初期化しているのに、x,yを後から読んでいる
vector<vector<int>> B(x, vector<int>(y, 0));  // x=0, y=0 のまま！
cin >> n;
// ...
cin >> x >> y;   // 遅い
cout << B[x-1][y-1];  // セグフォ
```

vectorのサイズは**使う前に確定**させる必要がある。
