# `next_permutation`

`next_permutation` は、配列や `vector` を辞書順で「次の並び」に並べ替える関数。

```cpp
next_permutation(v.begin(), v.end())
```

戻り値は `bool`。

- 次の並びがあれば `true`
- もう最後の並びなら `false`

## 典型形

```cpp
vector<int> v = {4, 5, 6};
do {
  // 今の並びを使う
} while (next_permutation(v.begin(), v.end()));
```

このコードで、

```txt
{4,5,6}
{4,6,5}
{5,4,6}
{5,6,4}
{6,4,5}
{6,5,4}
```

の 6 通りを全部なめられる。

## なぜ `do-while` なのか

`while` だと最初の `{4,5,6}` を使う前に `next_permutation` が動いてしまう。

今回ほしいのは「今の並びを使ってから次へ進む」なので `do-while` が自然。

## なぜ最初に昇順で置くのか

`next_permutation` は「今の並びより辞書順で次」を作る。
全部の順列を漏れなく回したいなら、最初は最小の並びにしておく。

```cpp
vector<int> v = {4, 5, 6};
```

または

```cpp
sort(v.begin(), v.end());
```

としてから始める。

## 今回の問題で何をしていたか

`v = {4,5,6}` は、

- 1 個目のサイコロにどの値を担当させるか
- 2 個目のサイコロにどの値を担当させるか
- 3 個目のサイコロにどの値を担当させるか

を表している。

たとえば `v = {4,6,5}` なら、

- 1 個目が 4
- 2 個目が 6
- 3 個目が 5

という割り当て。

`cnt[i][x]` を「`i` 番目のサイコロに `x` が書かれた面の数」とすると、

```cpp
cnt[0][v[0]] * cnt[1][v[1]] * cnt[2][v[2]]
```

はその割り当てで成功する面の組数になる。

## まとめ

- 「並び替えを全部試す」関数が `next_permutation`
- 全順列を回すときは昇順スタート
- 典型形は `do { ... } while (next_permutation(...));`
- 今回は `4,5,6` の 6 通りの割り当てを数えていた
